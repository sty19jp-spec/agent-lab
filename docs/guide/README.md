# Agent-Lab Guide Set

This guide set is a **TrackA / Layer5 completion snapshot** for the `agent-lab` repository.

It is intended as a private, shareable orientation package for a technically literate reader who needs to understand what the repository is, how it operates, and what has been completed at this snapshot point.

This guide set does **not** describe work beyond the TrackA Layer5 completed state.

## Guide files

- `docs/guide/agent-lab-guide.md`
  - Main narrative guide for the repository, architecture, workflow, and governance model.
- `docs/guide/setup-and-usage.md`
  - Practical setup, inspection, daily use flow, and executor entrypoint guidance.
- `docs/guide/safety-and-boundaries.md`
  - Merge-only governance, human boundaries, executor boundaries, and what is intentionally forbidden.
- `docs/guide/repository-map.md`
  - Short map of major top-level directories and what they are used for.

## Snapshot scope

This guide describes:

- what `agent-lab` is as of TrackA Layer5 completion
- how the current executor-centered PR workflow operates
- how validator-aware governance and merge-only delivery work
- what operators and reviewers should and should not do

This guide does not document later roadmap expansion. Ongoing development may continue after this snapshot, but that future work is outside the scope of this guide set.
